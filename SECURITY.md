# 보안·배포 시 참고

## 브라우저 저장소

- `localStorage` / `sessionStorage`는 **출처(프로토콜 + 호스트 + 포트)** 가 같을 때만 공유됩니다. 다른 사이트나 `file://` 로 열면 데이터가 **다릅니다**.
- 세션은 탭 단위 `sessionStorage`에 두어, 브라우저를 닫거나 탭을 끊으면 로그인 상태가 사라지도록 했습니다.

## 비밀번호

- 현재 데모는 브라우저에서 **PBKDF2(SHA-256, 10만 회)** 로만 해시합니다. **클라이언트만으로는 안전한 인증이 될 수 없습니다.**
- 실서비스에서는 **HTTPS** 위에서 **서버가** 가입·로그인을 처리하고, **bcrypt/Argon2** 등으로 해시·검증하고, 토큰(세션/JWT)을 발급하는 구조로 바꾸는 것이 맞습니다.
- `auth-config.js`의 `storageMode`를 `"api"`로 바꾼 뒤, API URL과 `auth.js`의 네트워크 레이어를 구현하면 같은 UI로 전환할 수 있게 두었습니다.

## 미리보기

- `Web Crypto`(PBKDF2)는 **보안 컨텍스트**에서 동작합니다. `http://127.0.0.1` / `http://localhost` 또는 **HTTPS**로 여세요. `file://` 직접 열기는 제한될 수 있습니다.
